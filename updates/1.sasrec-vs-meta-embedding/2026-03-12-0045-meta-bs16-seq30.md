# Meta batch_size 16 + seq_len 30 실험

## 가설

- meta champion 설정에서 `seq_len`을 `20`에서 `30`으로 늘리면 더 긴 사용자 이력을 활용해 benchmark 성능이 개선될 수 있다.

## 변경 내용

- 새 설정 파일 추가
  - `hm_refactored/configs/config.m1_local_meta_transformer_hash_benchmark_bs16_seq30.json`
- 단일 변경 축
  - `model_params.seq_len`: `20 -> 30`

## 실행 명령

```bash
source .venv/bin/activate
python hm_refactored/train.py --config_path hm_refactored/configs/config.m1_local_meta_transformer_hash_benchmark_bs16_seq30.json
```

## baseline vs treatment

### Baseline

- config: `config.m1_local_meta_transformer_hash_benchmark_bs16.json`
- best epoch: `2`
- best benchmark: `B_HR 0.0095`, `B_NDCG 0.0040`
- best test: `T_HR 0.0006`, `T_MAP 0.0001`

### Treatment

- config: `config.m1_local_meta_transformer_hash_benchmark_bs16_seq30.json`
- best epoch: `1`
- best benchmark: `B_HR 0.0101`, `B_NDCG 0.0051`
- best test: `T_HR 0.0006`, `T_MAP 0.0002`

## 해석

- `seq_len=30`은 meta benchmark 기준으로 `HR`, `NDCG`를 모두 개선했다.
- `T_MAP`도 baseline보다 소폭 나아졌다.
- 현재 메타 실험 축에서는 `batch_size=16`, `seq_len=30` 조합이 가장 좋은 결과다.
