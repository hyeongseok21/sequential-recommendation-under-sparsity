# Meta lr 2e-4 실험

## 가설

- metadata-on Transformer에서 학습률을 `1e-4`에서 `2e-4`로 올리면 최적화가 빨라져 benchmark 성능이 개선될 수 있다.

## 변경 내용

- 새 설정 파일 추가
  - `hm_refactored/configs/config.m1_local_meta_transformer_hash_benchmark_lr2e4.json`
- 단일 변경 축
  - `train_params.lr`: `1e-4 -> 2e-4`

## 실행 명령

```bash
source .venv/bin/activate
python hm_refactored/train.py --config_path hm_refactored/configs/config.m1_local_meta_transformer_hash_benchmark_lr2e4.json
```

## baseline vs treatment

### Baseline

- config: `config.m1_local_meta_transformer_hash_benchmark.json`
- best epoch: `1`
- best benchmark: `B_HR 0.0077`, `B_NDCG 0.0038`
- best test: `T_HR 0.0006`, `T_MAP 0.0003`

### Treatment

- config: `config.m1_local_meta_transformer_hash_benchmark_lr2e4.json`
- best epoch: `1`
- best benchmark: `B_HR 0.0089`, `B_NDCG 0.0036`
- best test: `T_HR 0.0006`, `T_MAP 0.0001`

## 해석

- `lr=2e-4`는 BPRLoss를 더 빠르게 낮췄다.
- 하지만 핵심 판단 기준으로 보면 `B_HR`는 소폭 올랐어도 `B_NDCG`와 `T_MAP`는 악화됐다.
- 현재 기준에서는 더 높은 학습률이 meta 성능 개선으로 이어졌다고 보기 어렵다.
