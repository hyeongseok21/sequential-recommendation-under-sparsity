# Meta batch_size 16 실험

## 가설

- metadata-on Transformer에서 `batch_size`를 `32`에서 `16`으로 줄이면 최적화가 더 안정화되어 benchmark 성능이 좋아질 수 있다.

## 변경 내용

- 새 설정 파일 추가
  - `hm_refactored/configs/config.m1_local_meta_transformer_hash_benchmark_bs16.json`
- 단일 변경 축
  - `train_params.batch_size`: `32 -> 16`

## 실행 명령

```bash
source .venv/bin/activate
python hm_refactored/train.py --config_path hm_refactored/configs/config.m1_local_meta_transformer_hash_benchmark_bs16.json
```

## baseline vs treatment

### Baseline

- config: `config.m1_local_meta_transformer_hash_benchmark.json`
- best epoch: `1`
- best benchmark: `B_HR 0.0077`, `B_NDCG 0.0038`
- best test: `T_HR 0.0006`, `T_MAP 0.0003`

### Treatment

- config: `config.m1_local_meta_transformer_hash_benchmark_bs16.json`
- best epoch: `2`
- best benchmark: `B_HR 0.0095`, `B_NDCG 0.0040`
- best test: `T_HR 0.0006`, `T_MAP 0.0001`

## 해석

- `batch_size=16`은 meta benchmark 기준으로 `HR`, `NDCG`를 모두 소폭 개선했다.
- 다만 `T_MAP`는 baseline보다 낮아졌다.
- 메타 설정 내부에서는 `batch_size=16`이 현재까지 가장 나은 benchmark 결과를 보였다.

## 판단

- gate verdict: PASS
- meta champion 교체: `config.m1_local_meta_transformer_hash_benchmark_bs16.json`
- 전체 local champion은 여전히 non-meta Transformer 유지
